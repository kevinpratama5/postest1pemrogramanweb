Blob URL — Shows Parent Origin in Address Bar

A blob URL created from https://accounts.google.com displays as blob:https://accounts.google.com/uuid. The user sees "accounts.google.com" — but content is 100% controlled by who created the blob (via XSS on Google). 

// If you have ANY code execution on accounts.google.com (XSS, etc.):
const html = '<h1>Fake Google Login</h1><form>Enter credentials: <input></form>';
const blob = new Blob([html], {type: 'text/html'});
const url = URL.createObjectURL(blob);
// url = "blob:https://accounts.google.com/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
window.open(url); // Shows accounts.google.com in address bar!

data: URI — No Origin (blank address bar)

data: URIs show no domain — just the raw data URL. On mobile, this can be confused for a trusted page especially if the page looks legitimate. Also used in phishing bypasses when filters only check domain names. 

// Navigate or open popup to data: URL
const page = btoa('<h1>Fake Login</h1><p>Enter your password:</p>');
window.open(`data:text/html;base64,${page}`);

Browser Behavior Matrix

BLOB URLs:
  Chrome:           Shows "blob:https://origin/uuid" — origin visible
  Firefox:          Shows "blob:https://origin/uuid" — origin visible
  Edge:             Same as Chrome
  Safari:           Same, blob: with origin
  Status: By design — but exploitable if attacker has XSS on trusted domain

DATA: URIs:
  Chrome (desktop): Shows full data: URI — obviously weird
  Chrome (mobile):  May show shortened version, less obvious
  Firefox (desktop): Shows full data: URI
  Firefox (Android): CVE-2016-1940 — could spoof with data: + address bar confusion
