What is IDN Homograph?

Internationalized Domain Names (IDN) allow Unicode characters in domain names. Many Unicode characters are visually identical to ASCII characters. An attacker registers a domain using lookalike chars — the browser displays it as the real domain, but it actually points to the attacker's server. 

Real:     google.com  (all ASCII)
Fake:     ɡoogle.com  (U+0261 LATIN SMALL LETTER SCRIPT G — looks identical!)
Punycode: xn--oogle-qmc.com  (what DNS actually sees)

Real:     apple.com
Fake:     аpple.com   (U+0430 CYRILLIC SMALL LETTER A — identical to ASCII 'a')
Punycode: xn--pple-43d.com

  → Buy xn--[generated].com on Namecheap/GoDaddy

STEP 3: Test in each browser
  → Desktop: Chrome, Firefox, Edge, Brave, Opera, Arc
  → Mobile: Chrome Android, Firefox Android, Samsung Internet
  → iOS: Safari, Chrome iOS, Firefox Focus

STEP 4: For each browser note:
  → Does address bar show Unicode (attack works) or Punycode (protected)?
  → Does any security indicator change (lock icon, color)?
  → Is there any warning shown to user?

STEP 5: If browser shows Unicode (attack works):
  → Document browser name and version
  → Screenshot address bar showing fake domain
  → Compare: does Chrome protect? What's the delta?
